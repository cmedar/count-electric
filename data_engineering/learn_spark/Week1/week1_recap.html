Driver
Runs your main(). Creates SparkContext. Builds the DAG, schedules stages and tasks. Holds the query plan. One per application.
Cluster Manager
Allocates resources (CPU, RAM) from the cluster. Spark is agnostic — works with YARN, Kubernetes, Standalone, Mesos, or Databricks-managed.
Executor
JVM process on a worker node. Runs tasks assigned by the driver. Stores cached data. Sends metrics back to driver.
SparkContext vs SparkSession
SparkContext — old RDD-era entry point.
SparkSession — modern unified entry point. Also exposes .sparkContext.

Narrow vs Wide Transformations

TypeDefinitionExamplesShuffle?
Narrow Each output partition depends on exactly one input partition filter, select, map, union, coalesce No
Wide Output partitions depend on multiple input partitions groupBy, join, distinct, orderBy, repartition Yes
Note: coalesce(n) reduces partitions without a full shuffle (narrow). repartition(n) does a full shuffle (wide). Use coalesce when reducing partition count.

Fault Tolerance & Lineage

RDD Lineage
Spark tracks the full chain of transformations (DAG). If a partition is lost, Spark recomputes it from its lineage — no data replication needed.
Checkpointing
Breaks the lineage by saving state to disk. Essential for long lineage chains (e.g., streaming) or graphs where recomputation would be too expensive.
Speculative Execution
If a task runs slower than others, Spark launches a duplicate on another node. First to finish wins. Controlled by spark.speculation.

Creating DataFrames

# From a list of tuples
df = spark.createDataFrame(
    [(1, "Alice", 29), (2, "Bob", 34)],
    schema="id LONG, name STRING, age INT"

# With explicit StructType
from pyspark.sql.types import StructType, StructField, StringType, LongType
schema = StructType([
    StructField("id",   LongType(),   nullable=False),

# Select
df.select("name", "age")
df.select(col("age") + 1)

# Filter  (where and filter are aliases)

df.where("age > 30")

# Add / rename / drop columns
df.withColumn("age_next", col("age") + 1)
df.withColumnRenamed("age", "years")

# Sort
df.sort(col("age").desc())
df.orderBy("name", ascending=False)

# Limit and distinct
df.limit(100)
df.distinct()

Joins

# Default join is inner

# Broadcast join — hint to avoid shuffle for small tables

Exam tip: df.join(other, on="id") with a string column name avoids column duplication. Using a condition like df.id == other.id creates two id columns — you must drop one.

Window Functions

from pyspark.sql import Window
from pyspark.sql.functions import rank, dense_rank, row_number, lag, lead, sum as _sum

w = Window.partitionBy("dept").orderBy(col("salary").desc())

df.withColumn("rank",        rank().over(w))
  .withColumn("dense_rank",  dense_rank().over(w))
  .withColumn("row_number",  row_number().over(w))
FunctionTiesGap after tie?Example (scores 100,100,90)

# Running sum (cumulative) within partition
w2 = Window.partitionBy("dept").orderBy("date").rowsBetween(Window.unboundedPreceding, Window.currentRow)
df.withColumn("cum_sales", _sum("sales").over(w2))

# Lag / Lead
df.withColumn("prev_sales", lag("sales", 1).over(w))
df.withColumn("next_sales", lead("sales", 1).over(w))

4 Phases
1. Analysis — resolve names & types
2. Logical Optimization — predicate pushdown, constant folding
3. Physical Planning — choose algorithms
4. Code Generation — compile to JVM bytecode (Tungsten)

Filters are pushed as early as possible — ideally into the data source scan (e.g., Parquet column pruning + row group skipping).

Reading & Writing

Reading Data

# CSV — infer schema (slower; scans full file)
df = spark.read.format("csv").option("header", "true").option("inferSchema", "true").load("path/")

# CSV — explicit schema (faster; production-safe)
df = spark.read.csv("path/", header=True, schema=my_schema)

# Parquet (schema embedded in file)
df = spark.read.parquet("path/")

# JSON
df = spark.read.json("path/")

# Delta
df = spark.read.format("delta").load("path/")

# Options chaining
df = (spark.read
    .format("csv")
    .option("header", True)
    .option("sep", ";")
    .schema(my_schema)
    .load("path/")
)

Writing Data

# Save modes
df.write.mode("overwrite").parquet("output/")
df.write.mode("append").format("delta").save("output/")

# Partition by column (creates directory per value)
df.write.partitionBy("year", "month").parquet("output/")

# Save as table (catalog registered)
df.write.saveAsTable("my_database.my_table")

# Control output file count
df.coalesce(1).write.csv("output/")   # single file (use with care)

Delta Lake Basics

# Convert Parquet to Delta
from delta.tables import DeltaTable
DeltaTable.convertToDelta(spark, "parquet.`path/`")

# Time travel
spark.read.format("delta").option("versionAsOf", 3).load("path/")
spark.read.format("delta").option("timestampAsOf", "2024-01-01").load("path/")

# History
DeltaTable.forPath(spark, "path/").history().show()

# Merge (upsert)
DeltaTable.forPath(spark, "target/").alias("t") \
  .merge(updates_df.alias("s"), "t.id = s.id") \
  .whenMatchedUpdateAll() \
  .whenNotMatchedInsertAll() \
  .execute()
Exam tip: Delta stores data as Parquet files + a transaction log (_delta_log/). ACID is achieved via the transaction log, not special file encoding.

Exam Tips & Common Traps

Architecture & Execution

  • Only actions trigger job execution. Transformations are lazy. Classic trap: "which of these triggers a job?" — pick count(), not filter().
  • Number of tasks in a stage = number of input partitions for that stage. After a shuffle, default post-shuffle partitions = spark.sql.shuffle.partitions (default 200).
  • coalesce is narrow (no shuffle); repartition is wide (shuffle). Only repartition can increase partition count reliably.
  • Driver runs in client mode on the submitting machine, cluster mode inside the cluster. Notebooks always use client mode.
  • Executor memory layout: spark.executor.memory → heap for storage + execution. spark.executor.memoryOverhead → off-heap (native libs, Python workers).

DataFrame API

  • select() returns only specified columns. withColumn() adds/replaces one column but keeps the rest.
  • filter() and where() are identical — both narrow transformations.
  • Joining on a string column name (df.join(other, "id")) avoids duplicate columns. Joining on a condition (df.id == other.id) keeps both — you must drop one.
  • explode() drops null or empty arrays. explode_outer() preserves rows with null/empty array as a single null-value row.
  • collect() pulls all data to the driver — dangerous on large datasets. Use take(n) or show(n) instead.

Spark SQL

  • Global temp views must be queried with the global_temp. prefix. Forgetting this prefix is a common wrong-answer trap.
  • rank() leaves gaps after ties (1,1,3). dense_rank() does not (1,1,2). row_number() is unique even for ties (arbitrary tiebreak).
  • Catalyst optimizer phase order: Analysis → Logical Optimization → Physical Planning → Code Generation. Predicate pushdown happens in Logical Optimization.
  • SQL DML: INSERT INTO appends. INSERT OVERWRITE replaces all data. MERGE INTO does upsert.

Reading & Writing

  • Default write mode is "error" (fails if path exists). You usually want "overwrite" or "append".
  • Parquet is columnar and splittable — the default format for analytics in Spark. CSV/JSON are row-based and slower for analytical queries.
  • inferSchema=True on CSV scans the whole file twice. In production, always provide an explicit schema.
  • Delta time travel: versionAsOf takes an integer version number; timestampAsOf takes a date/timestamp string.
  • Managed vs External tables: dropping a managed table deletes both metadata and data files. Dropping an external table deletes only the metadata.

Quick Decision Tree

Want to read faster from large files? → Use Parquet + predicate pushdown + column pruning
Join causing shuffle / OOM? → Broadcast the smaller table with broadcast() hint
Too many small output files? → coalesce(n) before writing
Need to re-use a DataFrame multiple times? → df.cache() or df.persist()
Common trap: df.cache() alone doesn't execute — it just marks the DF for caching. The first action that touches the DF will actually populate the cache.
