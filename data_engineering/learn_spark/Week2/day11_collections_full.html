Indexing trap: getItem(0) = first element (0-indexed). element_at(col, 1) = first element (1-indexed). element_at(col, -1) = last element.
Key array functions
Set operations
F.array_distinct("tags") # deduplicate F.array_union("a", "b") # union, no duplicates F.array_intersect("a", "b") # common elements F.array_except("a", "b") # in a but not in b
Math — quick reference
F.round("price", 2) F.bround("price", 2) # bround = banker's rounding F.ceil("x") F.floor("x") F.abs("x") F.greatest("a","b","c") # max across columns (same row) F.least("a","b","c") # min across columns (same row)
F.round(2.5) → 3. F.bround(2.5) → 2. Banker's rounding rounds to nearest even — use in financial pipelines.
Four variants — null behaviour is the exam question
explode silently drops rows with null or empty arrays. explode_outer keeps them as a null row. If your row count is off after explode, this is why.
posexplode — when you need the position index
df.select("id", F.posexplode("tags").alias("pos", "tag")) # pos is 0-indexed
inline — explode an array of structs into columns
df.select(F.inline("events")) # events: Array<Struct<ts:timestamp, type:string>> # result: one row per struct → columns ts | type
Inverse — re-collect rows back to array
df.groupBy("id").agg( F.collect_list("tag"), # keeps duplicates, order not guaranteed F.collect_set("tag") # deduplicates, order not guaranteed )
Neither collect_list nor collect_set guarantees element order. Apply F.array_sort() afterwards if order matters.
Creation
F.create_map(F.lit("k1"), F.col("v1"), F.lit("k2"), F.col("v2")) F.map_from_arrays(F.col("keys_arr"), F.col("vals_arr")) F.map_from_entries(F.col("arr_of_structs")) # [{key,value}] → map
Access — three equivalent ways
F.col("meta")["status"] F.element_at(F.col("meta"), "status") F.col("meta").getItem("status") # All return null if key missing — no KeyError
Inspection and manipulation
map_concat(a, b) merges two maps. On key conflict, the second map's value wins.
Structs — creation and access
F.struct(F.col("lat"), F.col("lon")).alias("coordinates") F.col("coordinates.lat") # dot notation F.col("address.city.district") # nested access
Which function to use?
from_json — full parse into typed struct
schema = StructType([StructField("user_id", IntegerType()), StructField("action", StringType())]) df.withColumn("p", F.from_json("payload", schema)) \ .select("p.user_id", "p.action")
get_json_object / json_tuple
F.get_json_object("payload", "$.address.city") # one field, nested path df.select(F.json_tuple("payload", "user_id", "action") .alias("uid", "act")) # multiple fields, one pass
from_json returns null silently on malformed JSON. Always filter nulls or route bad rows to a dead-letter table after parsing.
