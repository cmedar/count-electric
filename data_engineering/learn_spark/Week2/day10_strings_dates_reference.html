Day 10 reference: Spark string functions, date functions, higher-order functions, and quiz

Core string functions
Regex
regexp_replace
F.regexp_replace( "phone", r"[^0-9]", "" )
regexp_extract
F.regexp_extract( "email", r"@(.+)\.", 1 )
regexp_extract returns "" (empty string) on no match — not null. Exam trap when filtering on the result.
Parsing and formatting
string → date
F.to_date("col", "yyyy-MM-dd") F.to_timestamp("col", "yyyy-MM-dd HH:mm:ss")
date → string
F.date_format("col", "dd/MM/yyyy") F.date_format("col", "yyyy-MM")
Arithmetic
Extraction
F.year("d") F.month("d") F.dayofmonth("d") F.quarter("d") F.weekofyear("d") F.dayofweek("d") # 1=Sunday, 7=Saturday
Timezone
F.from_utc_timestamp("ts", "Europe/Bucharest") F.to_utc_timestamp("ts", "Europe/Bucharest") F.convert_timezone("UTC", "Europe/Bucharest", "ts") # Spark 3.4+
Delta stores timestamps as UTC. Never store local time in Delta if data crosses timezones. Convert at read time.
Higher-order functions on arrays
Spark SQL syntax
SELECT transform(prices, x -> x * 1.2) AS increased, filter(tags, x -> x LIKE 'de_%') AS de_tags, exists(scores, x -> x > 90) AS has_high, forall(scores, x -> x >= 0) AS all_positive, aggregate(prices, 0D, (acc,x) -> acc+x) AS total FROM products
Use higher-order functions instead of explode → groupBy → collect_list when you only need to transform or filter array elements in place.
